Introduction to Command Line

The Prompt

The prompt tells you that you are user sysadmin; the host or computer you are using: localhost; and the directory where you are at: ~, which represents your home directory.

sysadmin@localhost:~$

Show Username

The following command will display the same information that you see in the first part of the prompt.
The output of the whoami command, sysadmin, displays the user name of the current user. Although in this case your username is displayed in the prompt, this command could be used to obtain this information in a situation when the prompt did not contain this information.

sysadmin@localhost:~$ whoami                            
sysadmin                                                
sysadmin@localhost:~$

Information About the Current System

The next command displays information about the current system. To be able to see the name of the kernel you are using, type the following command into the terminal: uname
Many commands that are executed produce text output like this. You can change what output is produced by a command by using options after the name of the command. Options for a command can be specified in several ways. Traditionally in UNIX, options were expressed by a hyphen followed by another character, for example: -n.
In Linux, options can sometimes also be given by two hyphen characters followed by a word, or hyphenated word, for example: --nodename.

sysadmin@localhost:~$ uname                             
Linux 
sysadmin@localhost:~$ uname -n                          
localhost                                               
sysadmin@localhost:~$ uname --nodename                  
localhost

Print Working Directory

The pwd command is used to display your current "location" or current "working" directory.
The current directory in the example below is /home/sysadmin. This is also referred to as your home directory, a special place where you have control of files and other users normally have no access.
By default, this directory is named the same as your username and is located underneath the /home directory.

sysadmin@localhost:~$ pwd                                            
/home/sysadmin                                                       
sysadmin@localhost:~$

The Echo Command

The echo command can be used to print text, the value of a variable and show how the shell environment expands metacharacters.

sysadmin@localhost:~$ echo Hello World                             
Hello World                                                        
sysadmin@localhost:~$

Display the Value of the PATH Variable

The PATH variable is displayed by placing a $ character in front of the name of the variable. This variable is used to find the location of commands.
Each of the directories listed below are searched when you run a command.
If you try to run the date command, the shell will first look for the command in the /home/sysadmin/bin directory and then in the /usr/local/sbin directory and so on. Once the date command is found, the shell "runs it‚Äù.

sysadmin@localhost:~$ echo $PATH                                    
/home/sysadmin/bin:/usr/local/sbin:/usr/local/bin:/usr/sbin:/usr/bin:/sbin:/bin:/usr/games                                                 
sysadmin@localhost:~$

Find Out Location of the Command

Use the which command to determine if there is an executable file named date that is located in a directory listed in the PATH value.
The output of the which command tells you that when you execute the date command, the system will run the command /bin/date. The which command makes use of the PATH variable to determine the location of the date command.

sysadmin@localhost:~$ which date                                    
/bin/date                                                           
sysadmin@localhost:~$

Display All Filenames In the Current Directory Using Echo

The use of glob characters in Linux is similar to what many operating systems refer to as "wildcard" characters. Using glob characters, you match filenames using patterns.
The asterisk * matches "zero or more" characters in a file name. In the example above, this results in matching all filenames in the current directory.
The echo command, in turn, displays the filenames that were matched.

sysadmin@localhost:~$ echo *                                         
Desktop Documents Downloads Music Pictures Public Templates Videos   
sysadmin@localhost:~$

Display All Filenames In the Current Directory That Start With the Letter "D" and "P"

In the first example, D*, as "match all filenames in the current directory that begin with a capital d character and have zero or more of any other character after the D".

sysadmin@localhost:~$ echo D*                                        
Desktop Documents Downloads                                          
sysadmin@localhost:~$ echo P*                                        
Pictures Public                                                      
sysadmin@localhost:~$

Display All Filenames In the Current Directory That end in the letter "s"

The asterisk * can be used anywhere in the string.

sysadmin@localhost:~$ echo *s                                        
Documents Downloads Pictures Templates Videos                        
sysadmin@localhost:~$

Display Filenames Using the Asterisk Multiple Times

Notice that the asterisk can also appear multiple times or in the middle of several characters.

sysadmin@localhost:~$ echo D*n*s                                     
Documents Downloads                                                  
sysadmin@localhost:~$

Display Filenames Using Question Mark

The question mark matches exactly one character. This single character can be any possible character.
Like the asterisk it can be used anywhere in a string and can appear multiple times. Typing six of them will match six character filenames.

sysadmin@localhost:~$ echo ??????                                    
Public Videos                                                        
sysadmin@localhost:~$

Display Filenames Using Question Mark

Using the question mark with other characters will limit the matches. Type the following to display the file names that start with the letter D and are exactly nine characters long

sysadmin@localhost:~$ echo D????????                                 
Documents Downloads                                                  
sysadmin@localhost:~$

Display Filenames with Combination of Glob Characters

Wildcards or glob characters can be combined together. The following command will display file names that are at least six characters long and end in the letter "s".
Think of the pattern ?????*s to mean "match filenames that begin with any five characters, then have zero or more of any characters and then end with an "s" character".

sysadmin@localhost:~$ echo ?????*s                                   
Documents Downloads Pictures Templates Videos                        
sysadmin@localhost:~$

Display Filenames Using a Square Brackets

The next glob is similar to the question mark glob to specify one character.This glob uses a pair of square brackets [ ] to specify which one character will be allowed. The allowed characters can be specified as a range, a list, or by what is known as a character class. The allowed characters can also be negated with an exclamation point !. In the first example, the first character of the file name can be either a D or a P. In the second example, the first character can be any character except a D or P

sysadmin@localhost:~$ echo [DP]*                                     
Desktop Documents Downloads Pictures Public                          
sysadmin@localhost:~$ echo [!DP]*                                    
Music Templates Videos                                               
sysadmin@localhost:~$

The allowed characters can be specified as a range, a list, or by what is known as a character class. The allowed characters can also be negated with an exclamation point !. Display Filenames Using a Square Brackets With Range

In these next examples, a range of characters will be specified. In the first example, the first character of the file name can be any character starting at D and ending at P. In the second example, this range of characters is negated, meaning any single character will match as long as it is not between the letters D and P
The ASCII text table is used to determine the range of characters. You can view this table by searching for it on the Internet or typing the following command: ascii

sysadmin@localhost:~$ echo [D-P]*                                    
Desktop Documents Downloads Music Pictures Public                    
sysadmin@localhost:~$ echo [!D-P]*                                   
Templates Videos                                                     
sysadmin@localhost:~$

Executing Commands Using Back Quotes

To execute the date command within the line of the echo command, use back quotes `.

sysadmin@localhost:~$ echo Today is `date`                                      
Today is Sat Jan 19 15:59:28 UTC 2019                                           
sysadmin@localhost:~$

Executing Commands Using $ Sign And Brackets

You can also place $( before the command and ) after the command to accomplish command substitution.
Backquotes look very similar to single quotes, making it harder to "see" what a command is supposed to do. Originally shells used backquotes; the $(command) format was added in a later version of the Bash shell to make the statement more visually clear.

sysadmin@localhost:~$ echo Today is $(date)                                     
Today is Sat Jan 19 16:03:12 UTC 2019                                           
sysadmin@localhost:~$

Disabling Back Quotes

If you don't want the backquotes to be used to execute a command, place single quotes around them.

sysadmin@localhost:~$ echo This is the command '`date`'             
This is the command `date`                                           
sysadmin@localhost:~$

Disabling Back Quotes Another Way

Note that you could also place a backslash character in front of each backquote character.

sysadmin@localhost:~$ echo This is the command \`date\`              
This is the command `date`                                           
sysadmin@localhost:~$

Executing Command With Double Quotes

Double quote characters don't have any effect on backquote characters. The shell will still use them as command substitution. 

sysadmin@localhost:~$ echo This is the command "`date`"                         
This is the command Sat Jan 19 16:16:24 UTC 2019                                
sysadmin@localhost:~$

Double Quotes With Wildcard Characters

Double quote characters will have an effect on wildcard characters, disabling their special meaning.

sysadmin@localhost:~$ echo D*                                        
Desktop Documents Downloads                                          
sysadmin@localhost:~$ echo "D*"                                      
D*                                                                   
sysadmin@localhost:~$

Executing Multiple Commands In One Line

As you can see the output shows all three commands executed sequentially.

sysadmin@localhost:~$ echo Hello; echo Linux; echo Student           
Hello                                                                
Linux                                                              
Student                                                             
sysadmin@localhost:~$

Executing Multiple Commands In One Line With First Command Failure

Note that all three commands still executed even though the first one failed. While you can't see from the output of the false command, it did execute.
However, when commands are separated by the ; character, they are completely independent of each other.

sysadmin@localhost:~$ false; echo Not; echo Conditional          
Not                                                       
Conditional                                              
sysadmin@localhost:~$

Executing Multiple Commands In One Line Using "logical and"

Because each echo statement executes correctly, a return value of success is provided, allowing the next statement to also be executed.

sysadmin@localhost:~$ echo Start && echo Going && echo Gone          
Start                                                                
Going                                                                
Gone                                                                 
sysadmin@localhost:~$

Executing Multiple Commands In One Line Using "logical and" With Failure

The first echo command succeeds and we see its output. The false command executes with a failure result, so the last echo statement is not executed.

sysadmin@localhost:~$ echo Success && false && echo Bye              
Success                                                              
sysadmin@localhost:~$

Executing Multiple Commands In One Line Using "logical or" Statement

The "or" characters separating the following commands demonstrates how the failure before the "or" statement causes the command after it to execute; however, a successful first statement causes the command to not execute.

sysadmin@localhost:~$ false || echo Fail Or                         
Fail Or                                                              
sysadmin@localhost:~$ true || echo Nothing to see here                    
sysadmin@localhost:~$

Shell History

The first and easiest way to recall a previous command is to use the up arrow key. Each press of the up arrow key goes backwards one command through history. When you find the command that you want to execute, you can use the left arrow keys and right arrow keys to position the cursor for editing.
Another way to use your command history is to execute the history command to be able to view a numbered history list. The number listed to the left of the command can be used to execute the command again. The history command also has a number of options and arguments which can manipulate which commands will be stored or displayed.

sysadmin@localhost:~$ echo Hi
Hi
sysadmin@localhost:~$ history
    1 date
    2 clear
    3 echo Hi 
    4 history
sysadmin@localhost:~$

See a Limited Number of History Commands

To view a limited number of commands, the history command can take a number as a parameter to display exactly that many recent entries.

sysadmin@localhost:~$ history 5                                      
   185  false || Fail Or                                       
   186  false || echo Fail Or                                        
   187  true || echo Nothing to see here                             
   188  history                                        
   189  history 5                                                  
sysadmin@localhost:~$

See a Limited Number of History Commands

To execute a command again, type the exclamation point and the history list number. For example, to execute the 94th command in your history list, you would execute the following.

sysadmin@localhost:~$ !94                                                        
echo Today is `date`                                                            
Today is Sat Jan 19 16:50:23 UTC 2019                                           
sysadmin@localhost:~$
